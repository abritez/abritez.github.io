Media Queries

Introduction

Media Queries allows you to conditionally control your CSS based on a variety of factors which include the type of media your site is being viewed on, the width of your browser viewport, even environmental factors like lighting conditions (coming soon)

A media query consists of two components: a media type that describes the medium that they user is on when accessing the website and an expression that states the properties and values that need to be true in order to for the corresponding style sheet or style rules are applied, following the normal cascading rules

Media Types

The value of the "media" attribute specifies on what device the linked document will be displayed.

  • print: Used for printers
  • screen: Used for computer screens, tablets, smart-phones etc.
  • speech: Used for screenreaders that "reads" the page out loud

Setting this to "all" would indicate that the styling rules apply to all the devices

Media Properties

Below are some of the media features that CSS could adapt for. We designing for mobile and desktop, the most common features to adapt for are typically screen width and orientation.

Feature Value Description
width length The viewport width. This value will change as the user resizes their browser.
height length The viewport hieght. This value will change as the user resizes their browser.
device-width length The width of the device, such as a computer screen. This is a static value.
device-height length The height of the device, such as a computer screen. This is a static value.
orientation portorate || landscape The orientation of the viewport (landscape or portrait mode). This change as the user rotates their device. Mostly used in a mobile context
aspect-ratio ratio The ratio between the width and the height of the viewport
color interger The number of bits per color component for the output device
color-index interger The number of colors the device can display
monochrome length The number of bits per "color" on a monochrome (greyscale) device
resolution resolution The resolution of the output device, using dpi or dpcm
scan progressive || interlace The scanning process of the output device
grid length Whether the device is a grid or bitmap

Syntax Examples

Anatomy of a Media Query in CSS

@media not|only mediatype and (expressions) {
    CSS-Code;
}

Anatomy of a Media Query on Link tag

<link rel="stylesheet" media="mediatype and|not|only (expressions)" href="print.css">

Breakpoints

The best practice in building websites that adapt to all screen size is to start with the smallest one. As your screen width increases in width you could a breakpoint that adapts to the larger display

There are a couple schools of thought on how to define breakpoint. You could either look for standardized widths across devices like 320 (iPhone), 768px (iPad), etc, or you could set breakpoints that work best for your content.

Below are the breakpoints set by Twitter bootstrap, using a mobile first approach. Keep in mind that there is no "rule", so be sure to test your breakpoint scheme with your own content.

/*==================================================
=            Bootstrap 3 Media Queries             =
==================================================*/


    /*==========  Mobile First Method  ==========*/

    /* Custom, iPhone Retina */ 
    @media only screen and (min-width : 320px) {

    }

    /* Extra Small Devices, Phones */ 
    @media only screen and (min-width : 480px) {

    }

    /* Small Devices, Tablets */
    @media only screen and (min-width : 768px) {

    }

    /* Medium Devices, Desktops */
    @media only screen and (min-width : 992px) {

    }

    /* Large Devices, Wide Screens */
    @media only screen and (min-width : 1200px) {

    }
