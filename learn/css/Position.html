Positioning

Static

This is the default position type of an element. With this positioning, your block level elements will stack one ontop of the next. Properties such as right, left, top, and bottom, are not available to static elements. That means if you'd need to override the static property default with a type that supports it.

Relative

Elements that are positioned relative work nearly identical than those that are static. The difference is that an element that has a relative position could specifify an top, left, bottom, and right offset

An important thing to note is that even though you are moving the elements using offset properties, the elements siblings layout as if the element was in the original position. In the example below notice that the second box shifted down, however the 3 box has the same vertical position, not effected by the position of the second box.

Absolute

Elements that have a position of absolute could be placed anywhere. Similarly to relative elements, absolute elements adhere to top, bottom, left, and right positioning.

The example below has 4 boxes position at each corner of the container. Make sure to look at the CSS.

An important note to keep in mind is that the coordinats that are specified on an absolute positioned element are based on the parent element. Not the window.

In the example below we have nested the red box inside of the second grey box, and are moving it 10 pixels down from there. However, notice that the parent element has been set to 'relative'. This will not work when inside an element that is positioned as 'static', since static does not have a cooredinate system.

Using absolute positioning also allows you to anchor side of the rectagle. This results in a resizable flow for your element.

In the example below the abosolute positioned box has both left and right value specified in the CSS.

Fixed

Fixed elements share all of the same features as absolute elements, with the exception that the coordantes are based on the browser (viewport), and not the parent element. That means that fixed elements will not scroll, and stay 'fixed' on the screen.

Fixed header and footer

An important thing to keep in mind when using either fixed headers or footer is to make sure you add margins to either the top or bottom of your content so it doesn't get covered by the fixed element. Typically this mean, if your header is 50px high, then you want to add margin-top: 50px to your content div.
