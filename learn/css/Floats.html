Floats

Float were created in order to achive similar layout techniques that have been used in print design from many years.

A floated box is positioned within the normal flow, then taken out of the flow and shifted to the left or right as far as possible. Content may flow along the side of a float.

When you float an element it becomes a block box. This box can then be shifted to the left or right on the current line. The markup options are "float: left", "float: right" or "float: none".

In the code below we created a CSS rule that floats any Figure element to right if it contains a class name of "right" and left if it contains the class name "left". This way all you need to do to change layout is simply change the class name you use.

figure.right{
	float: right;
}

figure.left{
	float: right;
}

Clearing floats

Sometime you may run into an issue where the parent of your floating object is not acting as expected. This is a pretty common problem when working with floats

In the example below you will notice that there is a container div that has a red border, but it is not resizing to wrap around it's children as you might expect.

We could fix this problem by adding overflow:auto to the parent element.

Depending on what you are doing, using overflow:auto might cause a scrollbar if your content is extending the boundary of the container. For example, if you have a long unbreaking text (ie. long url text) or a large image that is bigger then the container a scrollbar will show.

One option is to use overflow:hidden, but that just means that the overflow text or image will be hidden. The other option is create a max-width for you image. There is also a CSS rule for breaking large words like URLS called, word-wrap: break-word. That might work in some cases.
