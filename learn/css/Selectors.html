Selectors

In order to apply a style to element(s) you must first indentify the element that you would like to stylize. We do this by using

Universal Selector (*{})

The universal selector applies CSS style to all the elements on the page.

In the example above we have overridden every element on the page to use a font-family of Arial, instead of the defualt Serif font.

Type Selector (element{})

HTML is comprised of tags, these tags denote their element type. By specifying a type, we are able to apply styles to any element on the page that matches our selector. That means, if there multiple elements that share that name on the page, then they will all be applied

In the example above, only elements on the page that are of type "<h2>" are targeted to override the default serif font-family to "Arial" and the color "red".

If the element being targeted contains child elements within that have their CSS property set to inherit, then those child element will take on the same properties.

ID Selector (#idName{})

An ID selector starts off by using a hash (#) and immedidiatly followed by an ID name that was specified to an element in your HTML

In the example below we have an HTML file with multiple elements. One of these elements has an id of "content". All children elements are inheriting the values.

Class Selector (.idName{})

A class selector starts off by using a dot (.) and immedidiatly followed by an class name that was specified to the element(s) in your HTML

Descendant Selector (element .idName{})

Sometimes you might want add an additional level of speficity to your styles. An item with a class name of "special" that lives in a footer, might look different than one that lives in your main content area.

In the example above your notice that elements with a class name of "special" show up differantly.

Child Combinator Selector (element > element)

Very similar to the descendant selector, child combinators and another level of speficity were only immediate children are targeted.

To specify a child combinator you need to add a ">" in between the parent and child selectors.

In the example above your notice that elements with a class name of "special" show up differantly.

General Sibling Combinator Selector (element ~ element)

The general sibling combinator (~) separates two selectors and matches the second element only if it follows the first element (though not necessarily immediately), and both are children of the same parent element

In the example above your notice that elements with a class name of "special" show up differantly.

Adjacent Sibling Combinator Selector (element + element)

The adjacent sibling combinator (+) separates two selectors and matches the second element only if it immediately follows the first element, and both are children of the same parent element.

In the example above your notice that only one adjacent operator is targeting an element. The other is failing because the sibling is not "immedialy" following the element on the left of the selector.

Attribute Selector (element[attribute="value"])

Most elements have a variety of attibutes that could be added inside of the tag. For example your "<a target="_blank">" or "<input type="text">". Using an attribute selector we could target only those elements that are of that type. This is really handy when dealing with inputs. Inputs have a type attribute that specify the type of form element it is. Its pretty obvious that the style consideration you make for a radio button and submit button would differ greatly.

In the example above your notice the difference on how both the button and input text field are handling text color.

Pseudo-class Selector (element:state)

These class offer additional discription on the specific state of your element. These might include things like ":disabled" or ":focus". There are also some states that are more location based like ":first-child" and ":nth-child(2)" which targets the 2nd instance of that element.

To specify a pseudo class you need to add a ":" followed by your state name, and make sure it is appended to your existing selector.

In the example below there are 2 examples of pseudo-class selectors. The first is a ":hover" attached to a DIV. The other is a textfield that is in a ":focus" state.
